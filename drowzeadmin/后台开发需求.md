# Drowze SDK 后台管理系统开发需求

## 1. 项目概述

### 1.1 项目背景
Drowze SDK 是一个疲劳驾驶检测 SDK，面向企业客户提供疲劳驾驶检测能力。随着设备数量的增长（每月万台设备），需要一个后台管理系统来管理许可证、监控设备状态、统计使用数据等。

### 1.2 项目目标
- 提供许可证的创建、管理和监控
- 实现设备绑定与管理
- 提供使用数据统计和分析
- 支持客户管理和账单生成
- 确保系统安全和稳定性

### 1.3 设计原则
- **安全性**：多重加密，防破解，防篡改
- **可扩展性**：支持万台设备的大规模部署
- **可靠性**：高可用性，数据备份
- **易用性**：界面友好，操作简单

### 2.2 核心模块

- **数据分析服务**：使用数据统计分析，生成报表
- **通知服务**：预警和通知，及时处理异常

### 2.3 内置临时 License 机制
- **概述**：每个 SDK 构建版本自带一个 15 天有效期的临时 License
- **特性**：
  - 有效期：15 天（不可续期）
  - 类型：试用 License（IS_TRIAL = true）
  - 功能：完整 SDK 功能，无设备数量限制
  - 用途：客户测试体验
- **SDK 内置验证**：
  - SDK 初始化时检查 License 状态
  - 剩余 7 天时触发警告回调（onTrialWarning）
  - 过期后拒绝初始化（onError: "Trial license expired"）
  - 每次检测前验证 License 有效性
- **对比正式 License**：
  - 临时 License：15 天、不可续期、内置在 SDK 中
  - 正式 License：无有效期限制、可续期、通过后台激活

## 4. API 接口设计

### 4.1 SDK 接口

#### 4.1.1 激活接口
- **URL**：`POST /api/v1/license/activate`
- **功能**：激活 SDK 并绑定设备
- **参数**：
  - `license_key`：许可证密钥
  - `device_id`：设备指纹
  - `device_info`：设备信息（厂商、型号、系统版本等）
  - `timestamp`：时间戳
  - `signature`：请求签名（防止篡改）
- **响应**：
  - `success`：激活是否成功
  - `message`：响应消息
  - `token`：授权令牌
  - `expires_at`：过期时间
  - `device_id`：绑定的设备ID

#### 4.1.2 验证接口
- **URL**：`POST /api/v1/license/verify`
- **功能**：验证 SDK 授权状态
- **参数**：
  - `license_key`：许可证密钥
  - `device_id`：设备指纹
  - `token`：授权令牌
  - `timestamp`：时间戳
- **响应**：
  - `valid`：验证是否通过
  - `message`：响应消息
  - `remaining_offline_days`：剩余离线天数
  - `expires_at`：过期时间

#### 4.1.3 续期接口
- **URL**：`POST /api/v1/license/renew`
- **功能**：延长 SDK 授权有效期
- **参数**：
  - `license_key`：许可证密钥
  - `device_id`：设备指纹
  - `token`：授权令牌
  - `timestamp`：时间戳
- **响应**：
  - `success`：续期是否成功
  - `message`：响应消息
  - `expires_at`：新的过期时间
  - `remaining_offline_days`：更新后的离线天数

### 4.2 管理接口

#### 4.2.1 许可证管理
- `GET /api/admin/licenses`：获取许可证列表（支持分页和筛选）
- `POST /api/admin/licenses`：创建新许可证
- `GET /api/admin/licenses/{id}`：获取许可证详情
- `PUT /api/admin/licenses/{id}`：更新许可证信息
- `DELETE /api/admin/licenses/{id}`：删除许可证
- `POST /api/admin/licenses/{id}/renew`：手动续期许可证
- `POST /api/admin/licenses/{id}/disable`：禁用许可证
- `POST /api/admin/licenses/{id}/enable`：启用许可证

#### 4.2.2 设备管理
- `GET /api/admin/devices`：获取设备列表（支持分页和筛选）
- `GET /api/admin/devices/{id}`：获取设备详情
- `DELETE /api/admin/devices/{id}`：解绑设备
- `POST /api/admin/devices/{id}/disable`：禁用设备
- `POST /api/admin/devices/{id}/enable`：启用设备
- `GET /api/admin/devices/{id}/logs`：获取设备使用日志

#### 4.2.3 客户管理
- `GET /api/admin/customers`：获取客户列表（支持分页和筛选）
- `POST /api/admin/customers`：创建新客户
- `GET /api/admin/customers/{id}`：获取客户详情
- `PUT /api/admin/customers/{id}`：更新客户信息
- `DELETE /api/admin/customers/{id}`：删除客户
- `GET /api/admin/customers/{id}/licenses`：获取客户的许可证列表

#### 4.2.4 统计分析

## 6. 安全设计

### 6.1 认证与授权
- **API 认证**：
  - 使用 API Key + HMAC 签名机制
  - 请求签名 = HMAC-SHA256(API Secret + 请求参数 + 时间戳)
  - 时间戳验证，防止重放攻击
  - API Key 定期轮换
- **管理后台认证**：
  - 使用 JWT 认证
  - 密码使用 bcrypt 哈希存储
  - 双因素认证（可选）
  - 登录失败次数限制
- **权限控制**：
  - 基于角色的权限管理
  - 细粒度权限控制
  - 权限审批流程
  - 权限变更审计
- **会话管理**：
  - token 有效期控制
  - 自动续期机制
  - 会话失效处理
  - 多设备登录管理

### 6.2 数据安全
- **传输加密**：
  - HTTPS + TLS 1.3
  - 证书定期更新
  - 安全密码套件
- **存储加密**：
  - 敏感数据 AES-256 加密存储
  - 密钥管理和轮换
  - 数据脱敏处理（IP 地址仅存储到日志，后台不展示）
- **数据备份**：
  - 定期自动备份
  - 异地备份
  - 备份加密
  - 备份恢复测试
- **数据销毁**：
  - 符合 GDPR 要求
  - 安全删除机制
  - 数据销毁审计

### 6.3 防攻击措施
- **请求签名**：防止请求篡改
- **频率限制**：
  - API 调用频率限制
  - 登录尝试次数限制
  - 防暴力攻击
- **IP 白名单**：
  - 管理后台 IP 白名单
  - API 访问 IP 限制
- **SQL 注入防护**：
  - 参数化查询
  - ORM 框架使用
  - 输入验证
- **XSS 防护**：
  - 输入验证
  - 输出编码
  - Content-Security-Policy
- **CSRF 防护**：
  - 令牌验证
  - 同源策略

### 6.4 审计日志
- **操作日志**：
  - 记录管理员所有操作
  - 操作人、时间、IP、操作内容
  - 日志不可篡改
- **访问日志**：
  - 记录 API 访问
  - 请求参数、响应结果、耗时
  - 异常访问标记
- **异常日志**：
  - 记录系统异常
  - 错误类型、发生时间、影响范围
  - 自动告警
- **合规审计**：
  - 满足数据保护要求
  - 审计报告生成
